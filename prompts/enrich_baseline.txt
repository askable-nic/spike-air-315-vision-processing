## Output Fields

Provide values for ALL of the following fields:

- **frame_description**: A concise paragraph describing the full screenshot — page layout, visible content, key UI elements, and overall context. Be specific about what application or website is shown.
- **description**: A clear, specific description of what the user is doing at this moment (e.g. "User clicks the 'Search Flights' button", "User scrolls through hotel search results"). If the existing description is already good, you may refine or keep it.
- **interaction_target**: The specific UI element involved — use its visible label, placeholder text, or role (e.g. "'Search' button", "'Departure date' input field", "'Flights' tab"). Leave empty string if no specific element is targeted.
- **page_title**: The page or application title visible in the browser tab, heading, or title bar. Leave empty string if not visible.
- **page_location**: The URL visible in the browser address bar. Leave empty string if not visible or not a web page.

## Rules

1. Only describe what you can actually see in the screenshot(s) — do not invent or assume
2. Be specific: use actual labels, text, and element names visible on screen
3. For frame_description, describe the full page context, not just the interaction area
4. If a cropped close-up is provided alongside the full screenshot, use it to identify the specific element more precisely
5. Keep descriptions concise but informative — one or two sentences each
